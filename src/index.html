What is TypeScript?

Typescript is about solving a problem “How hard it is to write large applications in JavaScript”

When JavaScript was being created, the language didn't have any large-scale application structuring concepts like classes or modules. It was entirely dynamically typed, which means there's no static type information that we could use for smart tools.

TypeScript is a typed superset of JavaScript. Any browser. Any host. Any OS. Open Source.

JS/TS usage statistic by 2022

Why should you choose TypeScript over JavaScript?

  1. TypeScript is more reliable
  2. TypeScript is more explicit
  3. TypeScript and JavaScript are practically interchangeable, so why not?

Does TypeScript have any drawbacks?

Types take more time to add them. This is arguably not a big deal, but it is an argument in favor of JavaScript. Therefore, you might not choose TypeScript for small projects and prototypes for own use.

Optional Chaining


          let x = foo?.bar.baz();

          //More plainly, that code snippet is the same as writing the following.

          let x = foo === null || foo === undefined ? undefined : foo.bar.baz();

Nullish Coalescing


          let x = foo ?? bar();

          //Again, the above code is equivalent to the following.

          let x = foo !== null && foo !== undefined ? foo : bar();

Private variables


          class A {
            private x: number;
            protected y: number;

            constructor(x: number, y: number) {
                this.x = x;
                this.y = y;
            }
          }
